Maven License Verifier Plugin

Information

The goals for the MLV are bound to their respective phases in the build lifecycle.

The default binding is to run the MLV at the test phase. If you like to change this you have to make an appropriate configuration.

Hint: Sometimes this does not work, cause at that time the dependencies have not been solved (Check this!)

<project>
  ...
  <build>
    <plugins>
      <plugin>
        <groupId>com.soebes.maven.plugins.mlv</groupId>
        <artifactId>maven-licenses-verifier-plugin</artifactId>
        <version>0.2-SNAPSHOT</version>
        <executions>
          <execution>
            <phase>..</phase>
            <goals>
              <goal>verify</goal>
            </goals>
          <execution>
        <exectutions>
      </plugin>
    </plugins>
  </build>
  ...
</project>

The most important thing you have to do first is to create one or more licenses.xml files which contain the license information.

Configuration

The most important thing you have to do is to create a licenses.xml file which comprises of the different categories and give the location of the licenses.xml file in the configuration of the MLV plugin.

<project>
  ...
  <build>
    <plugins>
      <plugin>
        <groupId>com.soebes.maven.plugins.mlv</groupId>
        <artifactId>maven-licenses-verifier-plugin</artifactId>
        <version>0.2-SNAPSHOT</version>
        <configuration>
          <!-- Optional you can put your configurations here -->
        </configuration>
      </plugin>
    </plugins>
  </build>
  ...
</project>
